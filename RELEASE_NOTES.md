# Jinjeop Line Signage v2.5.0

## 릴리즈 개요

v2.5.0은 장시간 무인운영 복구 능력과 설정·공지 관리 사용성을 함께 강화한 기능 릴리즈입니다. 중앙 열차 위치 화면을 운영의 기준 화면으로 유지하면서, 앱·렌더러·주변 위젯·설정 파일·업데이트 경로가 각각 독립적으로 실패를 감지하고 복구하도록 정리했습니다.

## 24시간 무인운영 안정성

- 메인 프로세스와 렌더러 heartbeat를 기록하고 렌더러 응답이 멈추면 캐시 무시 새로고침, 렌더러 재시작, 앱 재실행 순으로 단계적 복구를 수행합니다.
- 설치본에 `JinjeopLineSignageWatchdog` Windows 예약 작업을 포함했습니다. 앱 heartbeat가 장시간 멈추면 외부 watchdog이 앱을 다시 실행합니다.
- 업데이트·의도적 종료·종료 준비 시간에는 watchdog을 일시 정지해 정상 재시작 및 PC 운영 일정과 충돌하지 않도록 했습니다.
- 전체화면 사이니지 운영 중에는 디스플레이 절전을 차단하고, Windows 절전 복귀 시 공지 일정·열차 자동 갱신·주변 위젯 데이터를 다시 동기화합니다.
- 렌더러/GPU 장애 이력을 바탕으로 일시적인 GPU 안전 모드를 적용할 수 있도록 복구 경로를 보강했습니다.
- 날씨, 시간표, 24절기, 오늘의 한마디가 유효한 데이터를 받지 못하면 실패한 위젯만 숨깁니다. 중앙 열차 위치 웹화면은 계속 표시됩니다.

## 열차 위치 정보와 화면 보존

- 실제 운영 신호인 `POST /traininfo/traininfoUserMap.do` 완료 상태를 계속 기록합니다.
- POST가 지연되어도 중앙 SMSS 화면을 숨기거나 반복 새로고침하지 않고 현재 화면을 보존하도록 변경했습니다.
- 드래그 보정 중 예외가 발생해도 `mouseUp`과 선택 영역 정리가 항상 실행되도록 해 입력 고착 가능성을 줄였습니다.
- 렌더러 자체가 종료된 경우에만 해당 webContents를 복구하고, 단순 데이터 지연과 화면 프로세스 장애를 구분합니다.

## 설정·캐시·로그 내구성

- `config.json`, 시간표, 24절기, 오늘의 한마디 캐시를 임시 파일 쓰기와 파일 동기화 후 교체하는 원자적 저장 방식으로 변경했습니다.
- 기본 JSON이 손상되면 손상 파일을 격리하고 `.bak` 백업에서 자동 복구합니다.
- SMSS 진단 로그를 최대 크기와 보관 개수 기준으로 순환시켜 장기 운영 중 로그가 무한히 증가하지 않도록 했습니다.
- HTTP 리다이렉트 횟수와 응답 크기를 제한해 잘못된 외부 응답이 메모리와 네트워크 작업을 장시간 점유하지 않도록 했습니다.
- 공지 보존 폴더에서 더 이상 참조되지 않는 오래된 미디어를 정리합니다.

## 운영 시간과 자동 업데이트

- 기본 자동 업데이트 확인 시간을 `00:30`으로 조정했습니다.
- 기본 PC 종료 기준을 `01:30`으로 조정하고, 10분 전인 `01:20`부터 자동 업데이트와 배경 갱신을 보류합니다.
- 기본 자동 작업 재개 시간은 `04:30`입니다.
- 이전 기본값 `01:00 / 02:00 / 04:30`을 그대로 사용하던 설정은 새 운영 시간으로 자동 이전합니다.
- 예약 업데이트 확인이 일시적으로 실패하면 10분, 15분, 15분 간격으로 제한된 재시도를 수행합니다.

## 공지 관리 기능

- 공지 파일 추가일을 게시 시작일 기본값으로 저장하고, 날짜 형식과 실제 달력 날짜를 검증합니다.
- 자정이 지나면 재생 목록의 `게시 전`, `게시 중`, `게시 종료`, `기간 오류` 상태와 실제 재생 대상을 자동 갱신합니다.
- 동일한 파일을 여러 번 추가하지 않도록 중복 등록을 차단합니다.
- 재생 목록에 전체 파일 수와 현재 재생 가능한 파일 수를 표시합니다.
- 공지가 없을 때는 첫 파일을 바로 추가할 수 있는 빈 목록 안내를 표시하고, 운영 화면의 중복 빈 상태 문구를 제거했습니다.
- 삭제 버튼과 순서 변경 핸들에 파일별 접근성 이름을 추가했습니다.

## 설정 화면과 저장 UX

- 공지 관리창에 누락되어 있던 전용 `공지 저장` 버튼을 추가했습니다.
- 일반 설정, 위젯, 열차 위치 정보, 공지 설정의 하단 작업줄을 고정해 긴 설정창에서도 저장 버튼이 항상 보이도록 했습니다.
- 상단 도구 모음과 각 설정창에 `저장됨` / `저장 필요` 상태를 표시합니다.
- 일반·위젯·열차·공지 설정 저장 성공 시 완료 토스트를 표시하고, 실패 시 오류 토스트와 함께 설정창을 유지합니다.
- 저장되지 않은 변경 사항이 있는 상태에서 사이니지 모드를 시작하면 먼저 자동 저장합니다. 저장 실패 시 사이니지 전환을 중단합니다.
- 화면에 오래된 고정 버전이 잠시 표시되지 않도록 실제 패키지 버전을 불러오기 전에는 `확인 중`으로 표시합니다.
- 저장 버튼과 상태, 공지 빈 목록의 대비·간격·반응형 배치를 정리했습니다.
