# Jinjeop Line Signage v2.5.1

## 릴리즈 개요

v2.5.1은 일부 PC와 간헐적인 화면 로딩 상황에서 열차 위치 정보 자동 새로고침이 확대 2회를 끝내기 전에 멈추던 문제를 해결한 패치 릴리즈입니다. 자동 실행 속도보다 완료 신뢰성을 우선해 웹 화면 준비, 레이아웃 안정, 4호선 선택, 실제 확대 배율, 드래그 보정 결과를 순서대로 확인합니다.

## 원인과 수정

- 기존 자동 새로고침은 페이지 재로딩 구간까지 포함한 전체 작업 잠금이 없어, 시작 시 자동 보정이나 다른 새로고침과 타이밍이 겹칠 수 있었습니다. 이제 전체 과정을 단일 작업으로 직렬화하고 동시에 들어온 요청은 진행 중인 작업의 결과를 함께 기다립니다.
- 자동 실행이 다른 보정 작업과 겹치거나 일시적으로 실패하면 다음 6시간 주기까지 다시 시도하지 않는 경로가 있었습니다. 단계 내부 재시도에 더해 30초, 90초, 3분 간격의 제한 재시도를 추가했습니다.
- 확대 버튼은 180ms 간격으로 호출만 하고 실제 배율 변화를 확인하지 않아, 느린 PC나 페이지 실행 문맥이 바뀌는 순간에도 두 번 성공한 것으로 오판할 수 있었습니다. 각 클릭마다 현재 배율을 읽고 `1.000 → 1.125 → 1.250` 변화와 최대 배율 도달을 확인하도록 바꿨습니다.
- 웹 로딩 이벤트 직후 바로 보정을 시작하던 흐름에 보수적인 대기 시간을 추가했습니다. 4호선 선택값, 열차 지도와 확대 버튼 존재 여부, 웹 영역 크기가 연속으로 안정됐는지를 확인한 뒤 확대와 드래그를 진행합니다.
- 한 단계가 불완전하면 전체 보정 순서를 최대 3회 다시 확인합니다. 실행 중 예외가 발생해도 기존과 같이 드래그 `mouseUp`과 선택 영역 정리는 항상 수행됩니다.

## 공지 없음 모드 개선

- 공지가 없거나 게시 기간이 끝나 공지 영역이 숨겨진 경우, 열차 위치 정보 영역이 전체 너비로 전환된 뒤 크기가 안정될 때까지 기다립니다.
- 보정 직전에 화면 크기와 공지 숨김 상태가 다시 바뀌면 해당 시도를 완료로 처리하지 않고 안정된 상태에서 재시도합니다.
- 공지 표시 상태와 공지 없음 상태의 드래그 보정 프로필을 계속 분리해, 공지 없이 장기간 운영해도 전체 너비 기준 보정값이 적용됩니다.
- 격리된 빈 공지 설정으로 시작 자동 보정과 강제 자동 새로고침을 연속 실행하는 패키지 검증 경로를 추가했습니다.

## 진단과 회귀 방지

- 확대 1회마다 클릭 순서, 재시도 횟수, 변경 전·후 배율과 최대 배율을 SMSS 진단 로그에 기록합니다.
- 자동 새로고침 합류, 지연 재시도, 레이아웃 안정 여부, 공지 표시 모드와 단계별 완료 결과를 로그에서 확인할 수 있습니다.
- 자동 새로고침 타이머 콜백에서 예외가 발생해도 다음 실행 또는 제한 재시도가 반드시 예약되도록 정리했습니다.

## 검증

- `node --check`: `config-defaults.js`, `main.js`, `renderer/renderer.js`, `preload.js`
- `git diff --check`
- `npm.cmd run verify:maintenance`
- `npm.cmd run verify:reliability`
- 소스 실행 자동 새로고침 검증: 빈 공지 상태에서 시작 자동 보정 후 강제 자동 새로고침 완료
- 패키지 실행 검증: `dist\win-unpacked\Jinjeop Line Signage.exe --smoke-test`
- 패키지 자동 새로고침 검증: `dist\win-unpacked\Jinjeop Line Signage.exe --smoke-test --smoke-test-auto-refresh --smoke-test-duration-ms=30000`
- 자동 새로고침 검증에서 공지 숨김 프로필, 4호선 선택, 확대 `1.000 → 1.125 → 1.250`, 드래그 보정 완료를 진단 로그로 확인
