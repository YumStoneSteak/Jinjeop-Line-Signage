# 진접선 행선안내 사이니지

진접선 안내 사이니지를 Windows PC에서 무인 운영하기 위한 Electron 앱입니다.

## 현재 버전

v2.5.0

## v2.5.0 주요 변경

- 메인·렌더러 heartbeat와 외부 Windows watchdog을 추가해 앱 정지, 렌더러 응답 중단, 비정상 종료를 단계적으로 자동 복구합니다.
- 설정·캐시 JSON을 원자적으로 저장하고 백업에서 복구하며, 진단 로그는 크기를 제한해 장기 무인운영 중 손상과 디스크 증가를 줄였습니다.
- 열차 위치 정보 POST가 지연될 때 현재 화면을 보존하고, 날씨·시간표·24절기·오늘의 한마디 오류는 해당 위젯만 격리합니다.
- 자동 업데이트 시간을 `00:30`, 종료 준비 시작을 `01:20`, PC 종료 기준을 `01:30`, 자동 작업 재개를 `04:30`으로 정렬했습니다.
- 공지 게시 시작일 기본값과 자정 상태 전환을 보강하고, 중복 파일 등록 방지·재생 가능 개수·빈 목록 안내를 추가했습니다.
- 공지 설정창에 전용 저장 버튼을 추가하고 모든 설정창 하단 작업줄을 고정했습니다. 미저장 상태와 저장 성공·실패 토스트도 표시합니다.
- 사이니지 모드를 시작할 때 남아 있는 변경 사항을 먼저 저장하며, 저장 실패 시 화면 전환을 중단합니다.
- NSIS 설치 파일명은 `Jinjeop.Line.Signage.v2.5.0.exe` 형식으로 생성됩니다.

## v2.4.0 Changes

- Fixed the SMSS train location page so the real `POST /traininfo/traininfoUserMap.do` stream is monitored and recovered when it stalls.
- The status bar now shows the actual last successful train-info POST time instead of the injected in-page `alive` timer.
- The station-save line-4 correction no longer forces an unnecessary reload when the SMSS page is already active.
- NSIS installer files are generated as `Jinjeop.Line.Signage.v2.4.0.exe`.

## 설치 및 실행

GitHub Release에서 받은 설치 파일을 실행합니다.

```text
Jinjeop.Line.Signage.v2.5.0.exe
```

설치 후 시작 메뉴 또는 바탕화면 바로가기로 실행합니다.

## 운영 시간 설정

상단 메뉴에서 **일반 설정**을 엽니다.

- **자동 업데이트**: 켜져 있으면 지정된 업데이트 시간에 새 버전을 확인하고 설치합니다.
- **업데이트 시간**: 기본 `00:30`입니다. PC 종료 전에 다운로드·설치·재실행 시간을 확보합니다.
- **PC 종료 시간**: 기본 `01:30`입니다. 이 시간 10분 전부터 자동 작업을 보류합니다.
- **PC 부팅 시간**: 기본 `04:30`입니다. 이 시간 이후 자동 작업을 다시 재개합니다.

앱은 PC를 끄거나 켜지 않습니다. PC 종료/부팅 또는 자동 로그인은 운영 환경에서 별도로 구성해야 합니다.

## 자동 업데이트

- 설치형 앱에서만 동작합니다.
- GitHub Releases의 `latest.yml`, `.blockmap`, NSIS 설치 파일을 기준으로 업데이트합니다.
- 자동 업데이트는 설정한 업데이트 시간에만 설치까지 진행합니다.
- 자동 작업 보류 시간대 안에 업데이트 시간이 들어가면 자동 업데이트를 실행하지 않고 설정 화면에 경고를 표시합니다.
- 필요할 때 **업데이트 확인** 또는 **지금 설치** 버튼으로 수동 실행할 수 있습니다.

## 개발 및 빌드

의존성 설치:

```powershell
npm install
```

개발 실행:

```powershell
npm start
```

개발 확인용 BAT 실행:

```powershell
.\run-dev-check.bat
```

이 파일은 의존성이 없으면 설치하고, 유지보수 시간 검증을 통과한 뒤 개발 모드로 앱을 실행합니다.

NSIS 설치형 빌드:

```powershell
npm run build
```

또는:

```powershell
.\build-installer.bat
```

GitHub Release까지 배포:

```powershell
.\release-github.bat
```

직접 배포 절차와 GitHub Desktop 사용 방법은 [RELEASE_GUIDE.md](./RELEASE_GUIDE.md)를 참고합니다.

빌드 결과:

```text
dist\Jinjeop.Line.Signage.v2.5.0.exe
dist\Jinjeop.Line.Signage.v2.5.0.exe.blockmap
dist\latest.yml
```

- **일반 설정**: 화면 분할, 창 동작, Windows 시작 시 자동 실행, 운영 시간, 자동 업데이트, 관리자 옵션
- **위젯**: 좌측 정보 영역의 표시 항목, 순서, 역 정보, 날씨, 열차 시간표
- **열차 위치 정보**: 시작 URL, 확대비율, 팝업 처리, 자동 새로고침, 드래그 보정
- **공지**: 이미지/영상 추가, 재생 시간, 게시 기간, 순서 변경
- **사이니지 모드 시작/종료**: 운영 화면과 설정 가능 화면 전환

## 운영 참고

- Windows 시작 시 자동 실행과 앱 시작 시 전체화면 모드는 기본으로 켜져 있습니다.
- 설치본은 `JinjeopLineSignageWatchdog` 예약 작업으로 앱 heartbeat를 감시하며, `01:20~04:30`에는 종료/부팅 절차와 충돌하지 않도록 재실행하지 않습니다.
- 완전 무인 운영을 하려면 Windows 자동 로그인, 전원 복구, PC 종료/부팅 정책은 운영 PC에서 별도로 설정해야 합니다.
- 날씨·시간표·24절기·오늘의 한마디가 유효한 데이터를 받지 못하면 해당 위젯만 숨깁니다. 중앙 열차 위치 웹화면은 POST가 끊겨도 숨기거나 덮지 않고 현재 화면을 유지합니다.
- 앱 설정은 Electron `userData` 경로의 `config.json`에 저장됩니다.
- Git에는 운영 PC의 개인 설정, 로그, 캐시 파일을 포함하지 않습니다.
