# 진접선 행선안내 사이니지

진접선 안내 사이니지를 Windows PC에서 무인 운영하기 위한 Electron 앱입니다.

## 현재 버전

v2.3.1

## v2.3.1 주요 변경

- 첫 설치 후 현재 역명을 선택하고 저장한 직후, 위젯 설정창이 닫힌 다음 열차 위치 정보 화면의 새로고침, 4호선 선택, 확대, 드래그 보정을 한 번 더 실행하도록 수정했습니다.
- 필수 역명 저장 시점에 설정 패널 레이아웃이 아직 열려 있어 드래그 보정 위치가 어긋날 수 있던 문제를 보완했습니다.
- 테스트/실행 환경의 출력 파이프가 닫힐 때 SMSS 진단 로그 출력 때문에 `EPIPE` JavaScript 오류 팝업이 뜰 수 있던 문제를 방지했습니다.
- `--smoke-test` 실행 시 사용자 화면에 흰 창을 띄우지 않고 숨김 상태로 검증 후 종료하도록 수정했습니다.
- NSIS 설치 파일명은 `Jinjeop.Line.Signage.v2.3.1.exe` 형식으로 생성됩니다.
- 바탕화면 바로가기는 `진접선 행선안내 사이니지 v2.3.1` 형식으로 생성됩니다.

## v2.1.0 주요 변경

- GitHub Releases 기반 자동 업데이트를 추가했습니다.
- 기본 자동 업데이트 시간은 `01:00`입니다.
- 기본 PC 종료 시간은 `02:00`, 기본 PC 부팅 시간은 `04:30`입니다.
- 앱은 Windows 종료/부팅을 제어하지 않습니다. 세 시간은 앱이 자동 작업 가능 여부를 판단하는 기준으로만 사용합니다.
- `02:00~04:30` 구간에는 자동 업데이트, 날씨/시간표/절기/오늘의 한마디 자동 갱신, 열차 위치 정보 자동 새로고침을 보류합니다.
- 사용자가 설정 화면에서 업데이트 시간, PC 종료 시간, PC 부팅 시간을 직접 수정할 수 있습니다.
- 수동 `업데이트 확인`, `지금 설치` 버튼을 추가했습니다.
- 배포 산출물은 NSIS 설치형만 유지합니다.
- 개발용 `run-dashboard*.bat`, local 복사 스크립트, portable 빌드 스크립트, 개발 watchdog 파일을 제거했습니다.

## 설치 및 실행

GitHub Release에서 받은 설치 파일을 실행합니다.

```text
Jinjeop.Line.Signage.v2.3.1.exe
```

설치 후 시작 메뉴 또는 바탕화면 바로가기로 실행합니다.

## 운영 시간 설정

상단 메뉴에서 **일반 설정**을 엽니다.

- **자동 업데이트**: 켜져 있으면 지정된 업데이트 시간에 새 버전을 확인하고 설치합니다.
- **업데이트 시간**: 기본 `01:00`입니다. PC 종료 시간 전에 업데이트가 끝나도록 이 값을 조정합니다.
- **PC 종료 시간**: 기본 `02:00`입니다. 이 시간부터 자동 작업을 보류합니다.
- **PC 부팅 시간**: 기본 `04:30`입니다. 이 시간 이후 자동 작업을 다시 재개합니다.

앱은 PC를 끄거나 켜지 않습니다. PC 종료/부팅 또는 자동 로그인은 운영 환경에서 별도로 구성해야 합니다.

## 자동 업데이트

- 설치형 앱에서만 동작합니다.
- GitHub Releases의 `latest.yml`, `.blockmap`, NSIS 설치 파일을 기준으로 업데이트합니다.
- 자동 업데이트는 설정한 업데이트 시간에만 설치까지 진행합니다.
- 자동 작업 보류 시간대 안에 업데이트 시간이 들어가면 자동 업데이트를 실행하지 않고 설정 화면에 경고를 표시합니다.
- 필요할 때 **업데이트 확인** 또는 **지금 설치** 버튼으로 수동 실행할 수 있습니다.

## 개발 및 빌드

의존성 설치:

```powershell
npm install
```

개발 실행:

```powershell
npm start
```

개발 확인용 BAT 실행:

```powershell
.\run-dev-check.bat
```

이 파일은 의존성이 없으면 설치하고, 유지보수 시간 검증을 통과한 뒤 개발 모드로 앱을 실행합니다.

NSIS 설치형 빌드:

```powershell
npm run build
```

또는:

```powershell
.\build-installer.bat
```

GitHub Release까지 배포:

```powershell
.\release-github.bat
```

직접 배포 절차와 GitHub Desktop 사용 방법은 [RELEASE_GUIDE.md](./RELEASE_GUIDE.md)를 참고합니다.

빌드 결과:

```text
dist\Jinjeop.Line.Signage.v2.3.1.exe
dist\Jinjeop.Line.Signage.v2.3.1.exe.blockmap
dist\latest.yml
```

## 주요 메뉴

- **일반 설정**: 화면 분할, 창 동작, Windows 시작 시 자동 실행, 운영 시간, 자동 업데이트, 관리자 옵션
- **위젯**: 좌측 정보 영역의 표시 항목, 순서, 역 정보, 날씨, 열차 시간표
- **열차 위치 정보**: 시작 URL, 확대비율, 팝업 처리, 자동 새로고침, 드래그 보정
- **공지**: 이미지/영상 추가, 재생 시간, 게시 기간, 순서 변경
- **사이니지 모드 시작/종료**: 운영 화면과 설정 가능 화면 전환

## 운영 참고

- Windows 시작 시 자동 실행과 앱 시작 시 전체화면 모드는 기본으로 켜져 있습니다.
- 완전 무인 운영을 하려면 Windows 자동 로그인, 전원 복구, PC 종료/부팅 정책은 운영 PC에서 별도로 설정해야 합니다.
- 앱 설정은 Electron `userData` 경로의 `config.json`에 저장됩니다.
- Git에는 운영 PC의 개인 설정, 로그, 캐시 파일을 포함하지 않습니다.
